# Test Plan - Swag Labs Manual Testing

## 1. Test Plan Identifier
SWAG-TP-01

## 2. Introduction
- Project: Swag Labs E-commerce Demo
- Objective: training and experience 
- Scope: Manual testing of the website's core functions through the UI using a brave browser

## 3. Test Items
 Login, Products, Cart, Checkout 

## 4. Features to be Tested / Not Tested

### Features to be tested 
login : valid login data || Invalid login data
product: sort || add to cart || remove from cart || preview the product 
cart : remove from cart || preview the product 
checkout : valid information || Invalid information || total price || cancel || finish 

### Not tested
server performance
security 

## 5. Approach
- Test Techniques: Black-Box (Decision Table) + Experience-based
- Test Levels: System Testing 

## 6. Test Deliverables
Test Cases, Bug Reports, Test Summary Report ,screenshots 

## 7. Schedule
Day 1: Test Plan & Test Cases 
Day 2: Execution & Bugs

## 8. Risks and Mitigation
 Risk: Login credentials change → Mitigation: Use documented users

## 9. Entry / Exit Criteria
Entry : website is on , have information login 
Exit Criteria : all test cases have been tested
